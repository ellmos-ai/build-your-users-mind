# build-your-users-mind

> **What you mind is what you get.**

**🌐 [EN](README.md) · [DE](README_de.md) · [ES](README_es.md) · [JA](README_ja.md) · [RU](README_ru.md) · [ZH](README_zh.md)** — 英語が公式であり、翻訳が遅れる場合があります。

AIエージェント（Claude、Codex、Gemini/agy、Kimiなど）が、自身の対話ログから、経験的かつ自己改善的な**theory-of-mind model of its own user**（ユーザー自身の心の理論モデル）を構築し、ユーザーの不在時にもその意図に沿って行動するためのレシピ。

**feedforward**によって動作します。ユーザー不在時に届くことのないフィードバックを待つ代わりに、エージェントは*ユーザーのフィードバックが届く前にそれを予測*し、その予測をガイダンス/報酬シグナルとして使用します。その後、予測と現実を評価して精度を向上させます。

## 「あなたの望みは分かっています」

この一言がすべてを物語っています。エージェントはユーザーの過去のプロンプトを読み取り、**ユーザーが何を決定したか、どのように表現したか、満足したかどうか**を抽出し、意思決定が必要でユーザーに連絡が取れない瞬間に参照できる一連の生きたドキュメントを作成します。

これはチャットボットのペルソナでも、重厚なフレームワークでもありません。メソッドと、いくつかのスクリプト、そしてドキュメントテンプレートで構成されています。エージェント固有の部分は、各エージェントが自身のログを読み込むための *source adapter* のみです。それ以外はすべて共通です。

## 仕組み — feedback precognition

0〜4のランタイムループ（`templates/START.md` を参照）：

| ステップ | ファイル | 役割 |
|---|---|---|
| 0 | プロジェクト `DECISIONS.md` | プロジェクト固有の決定が優先される（より具体的） |
| 1 | `WHAT-<USER>-SAID` | **証拠に基づく** ルール/決定（プロンプトIDの引用を含む） |
| 2 | `WHAT-WOULD-<USER>-SAY` | **precognition** (予知) — 予測されるフィードバック + 信頼度 (🟢/🟡/🔴) |
| 3 | `WHAT-I-DID…` + `MY-ACTIONS.txt` | 予測に基づいて実行されたアクションのログ |
| 4 | `WHAT-<USER>-SAID-ABOUT…` | **評価** — 予測 vs 現実 → (1) と (2) の改善 |

品質指標 = **予想された反応が、その後のユーザーの実際のフィードバックとどれだけ一致しているか。**
🔴（新規/パターンなし）の場合、ルールは**推測せず、エスカレーションする**ことです。

### パイプライン（モデルの構築）

1. **抽出 (Extract)** (`scripts/corpus_extract.py`) — 決定論的：ログから人間が入力したプロンプトのみを抽出し、合成されたターンをフィルタリングし、**機密情報を隠蔽**して、各プロンプトを次のターンの `outcome_signal`（称賛/修正/再発行/なし）にリンクします。
2. **分割 (Chunk)** (`scripts/chunk_corpus.py`) — 重複排除、オプションのドメイン、スウォーム（群れ）用のサイズ分割。
3. **分類 (Classify)** (スウォーム) — 8種類の分類（`TAXONOMY.md`） + decision_kind + 表現パターン。階層的なスウォーム（ドメインリーダー × チャックワーカー）。同梱されている `skills/swarm-operations/` を参照。
4. **集計 (Aggregate)** (`scripts/aggregate_stats.py`) — タイプ分布、B:K比率、転換点。
5. **作成 (Author)** — `templates/` からアバターファイルを作成し、エージェント自身のメモリ/ルールファイル（Claudeは `CLAUDE.md`、Codexは `GPT.md`/`AGENTS.md`、Geminiは `GEMINI.md` など）に短いポインタを**バインド (Bind)** します。

完全なレシピについては `SKILL.md` を、エージェントごとのログの場所については `SOURCE-ADAPTERS.md` を参照してください。

## Theory of us — 理論的背景

システムは、ユーザーを単独でモデル化するのではなく、**ダイアド**（エージェント ↔ ユーザー）をモデル化します。つまり、「私たちの理論（theory of us）」です。
以下に基づいています：
- LLMエージェント向けの **Theory of Mind**（心の理論）研究 — 対話相手の精神状態を予測し適応させることで、結果が改善される（例：*ToM-SWE*, arXiv 2510.21903; *Infusing Theory of Mind into Socially Intelligent LLM Agents*, 2509.22887; *Persistent Memory & User Profiles*, 2510.07925）。
- **Prompt-Archaeology**（プロンプト考古学、L. Geiger） — 人間とAIの完全なインタラクションプロトコルを分類する手法で、このモジュールはその8タイプの分類を再利用しています（`TAXONOMY.md`）。
- 既知の限界：LLMの心の理論は**繰り返されるケースでは堅牢ですが、新しい、または敵対的な変化に対しては脆弱**です。そのため、信頼度層と「推測せずエスカレーションする」ルールが存在します。

## バイアスと限界（信頼する前にお読みください）

- **暗黙の承認は不可視** — ユーザーは称賛ではなく修正を入力するため、モデルは修正を過大に表現し、「批判的」に偏る傾向があります。それに応じて調整してください。
- **エビデンスIDはLLMの要約から生成される** — 重要な決定的なものについては、生のコーパスと照らし合わせて検証してください。
- **分類器のバイアス** — サンプルをスポットチェックし、本格的な運用の場合は評価者間の一致率（inter-rater agreement）を報告してください。

## プライバシーと秘匿化

エクストラクターは、書き込みを行う**前**に、APIキー、トークン、メールアドレス、IPアドレスに類似したもの、および長い数列を隠蔽します。
**健康や税金、その他の機密性の高いユーザーコンテンツを隠蔽することはエージェントの責任**であり、コーパスやアバターファイルがプライベートな空間から出る前に対処する必要があります。本物のコーパスは絶対にコミットしないでください（`.gitignore` を参照）。
