# i18n — Translation Concept (build-your-users-mind)

> English is authoritative. Translations live cleanly separated under the `locales/` folder.

**Current status (2026-07-15):** the root English documents implement the `1.1.0-dev` safety and
pipeline contract. All `de/es/ja/ru/zh` documents are historical pre-1.1 drafts and are explicitly
marked as potentially stale. Do not use a translation as an operational or security contract until
its status is changed to current here.

## 1. Languages (6)
Like the ellmos MCP servers: **`en`** (source, authoritative, in root), **`de`**, **`es`**, **`ja`**, **`ru`**, **`zh`** (in `locales/<lang>/`).

## 2. Structure decision: language folders under `locales/`

To keep the repository root clean, all translated documents are organised in a `locales/` directory with per-language subfolders:

```
README.md             ← English, AUTHORITATIVE (root)
SKILL.md              ← English, AUTHORITATIVE (root)
...
locales/
  de/
    README.md         ← German translation
    SKILL.md          ← German translation
    ...
  es/
  ja/
  ru/
  zh/
```

- **Rationale:** because not only `README.md` but the whole set of user-facing documents (`SKILL.md`, `TAXONOMY.md`, `SOURCE-ADAPTERS.md`, `SECURITY.md`, `RELEASE_GATE.md`, `TODO.md`) is translated, a structured directory (`locales/<lang>/`) is worthwhile.
- **Language-switcher row** at the top of each `README.md` links the language versions via relative paths.
- **Disclaimer** at the top of every translation: *"English is authoritative; this translation may lag."* (rendered in the respective target language).

## 3. What is translated?
- All user- and developer-facing Markdown files from the root: `README.md`, `SKILL.md`, `TAXONOMY.md`, `SOURCE-ADAPTERS.md`, `SECURITY.md`, `RELEASE_GATE.md`, `TODO.md`.
- Source code and scripts (`scripts/`) as well as templates (`templates/`) and the bundled skill (`skills/`) remain English-only (authoritative).

## 4. Maintenance
- **Change EN first** → regenerate translations (e.g. via agy/Gemini).
- **Genuine diacritics + native scripts** (CJK/Cyrillic): the translating agent writes the files directly to the filesystem to avoid CJK corruption over stdout.

## 5. Markdown fidelity
Only **prose** is translated. Left unchanged: structure/heading anchors, links, code blocks, table scaffolding, proper nouns (`build-your-users-mind`, `feedforward`, file names, CLI flags, topics) and the tagline "what you mind is what you get".
