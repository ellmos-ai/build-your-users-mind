# i18n — Übersetzungskonzept (build-your-users-mind)

> Folgt dem **ellmos-MCP-Muster** (kein paralleler Standard). Englisch ist maßgeblich.

## 1. Sprachen (6)
Wie die ellmos-MCP-Server (`src/i18n/`): **`en`** (Quelle, maßgeblich), **`de`**, **`es`**, **`ja`**, **`ru`**, **`zh`**.

## 2. Struktur-Entscheidung: Suffix-Dateien, KEINE Sprach-Ordner

**Frage:** jede Sprache ein eigener Ordner? → **Nein.** Die MCP-Repos nutzen **Suffix-Dateien** im Root:

```
README.md          ← Englisch, AUTORITATIV
README_de.md
README_es.md
README_ja.md
README_ru.md
README_zh.md
```

- **Begründung:** Genau so machen es `ellmos-filecommander/-codecommander/-controlcenter/-clatcher` (`README.md` + `README_de.md`). Sprach-**Ordner** (`docs/<lang>/`) lohnen erst, wenn ein ganzer Doku-Baum übersetzt wird — hier nicht der Fall.
- **Sprachwähler-Zeile** oben in `README.md` verlinkt alle Versionen (Flagge/Label-Reihe, wie bei ellmos).
- **Disclaimer** oben in jeder Übersetzung: *„English is authoritative; this translation may lag."* (ellmos-Wortlaut).

## 3. Was wird übersetzt?
- **Nur die `README.md`** (das öffentliche Gesicht) — wie bei den MCP-Servern.
- **NICHT** übersetzt (bleiben Englisch, maßgeblich): `SKILL.md`, `SOURCE-ADAPTERS.md`, `TAXONOMY.md`, `scripts/`, `templates/`. Agenten lesen Englisch; das vermeidet 6× Pflegelast. (Optional später: `SKILL.md`.)

## 4. Pflege
- **EN zuerst** ändern → Übersetzungen neu generieren (Agent, z. B. agy/Gemini, via `_prompts/translate-readme.md`).
- **Echte Diakritika + native Schrift** (CJK/Kyrillisch): der übersetzende Agent muss die Dateien **direkt schreiben** (z. B. agy mit `--add-dir`), NICHT über stdout durchreichen — sonst CJK-Verstümmelung (U+FFFD). Siehe Übersetzungsprompt.
- Optional `last_synced`-Marker im Übersetzungs-Header.

## 5. Markdown-Treue
Übersetzt wird **nur Prosa**. Unverändert bleiben: Struktur/Überschriften-Anker, Links, Code-Blöcke,
Tabellen-Gerüst, Eigennamen (`build-your-users-mind`, `feedforward`, Dateinamen, CLI-Flags, Topics) und die Tagline „what you mind is what you get".
