# TAXONOMY — 8 типов запросов + классификационные поля

> Автономная версия (модуль является самодостаточным). **Методологическая основа:** *Prompt-Archaeology* (L. Geiger) — метод классификации полного протокола взаимодействия человека и ИИ.

## 8 типов запросов

| Тип | Код | Определение | Индикатор |
|---|---|---|---|
| Начальный запрос | **SP** | Инициирует новый анализ или фазу | Нет ссылок на предыдущий контекст |
| Уточняющий (тема) | **NT** | Углубляет существующую тему | «А как насчет...?» и т.п. |
| Уточняющий (метод) | **NM** | Запускает метод/инструмент/обзор/поиск/агента | Глагол действия |
| Уточняющий (управление) | **NS** | Управляет последовательностью или приоритетом | «Подожди», «сначала», «стоп» |
| Исправление | **KO** | Исправляет ошибку или предположение | Отрицание, контрпример |
| Подтверждение | **BE** | Проверяет промежуточное состояние | Краткое согласие/подтверждение |
| Смена курса | **RA** | Фундаментальное изменение направления | Ставит под сомнение все рамки работы |
| Мета-запрос | **MP** | О самом процессе или диалоге | Терминология процесса |

**Граничные случаи:** SP против NT (новый против связанного) · NM против NS (запуск метода против простого изменения порядка) · BE против KO («да, но...» обычно является KO) · RA встречается реже, чем KO, и касается всех рамок работы.

## Классификационные поля (на один запрос)

| Поле | Значения |
|---|---|
| `type_code` | SP/NT/NM/NS/KO/BE/RA/MP |
| `topic` | Краткая тема (связанная с проектом) |
| `is_decision` | true, если это решение, предпочтение, правило, исправление или смена курса |
| `decision_kind` | preference / correction / rule / direction_change / approval / rejection / process / none |
| `formulation_pattern` | Характерная формулировка пользователя (оригинальное выражение, кратко) |
| `method_triggered` | WebSearch / WebFetch / Multi-Agent / Review / Cross-Model / Script / LaTeX / -- |
| `is_turning_point` | true/false |
| `outcome_signal` *(детерминировано, этап 0/1)* | praise / correction / reissue / none (извлекается из следующего шага пользователя) |

## Индикаторы предвзятости (этап 4)
- **Подтверждение:Исправление (B:K)** — Дисбаланс указывает на предвзятость подтверждения; **молчаливое одобрение невидимо** (не печатается) → исправления перепредставлены.
- **Частота исправлений по темам** — Темы, склонные к ошибкам.
- **Проактивный:Реактивный** — Ведет ли пользователь диалог или диалогом управляет ИИ?
- **Частота смены курса** — Эпистемическая гибкость.
