# Политика безопасности

## Сообщения об уязвимостях
Пожалуйста, сообщайте об уязвимостях с помощью функции **GitHub Private Vulnerability Reporting** в этом репозитории (Security → Report a vulnerability). Не создавайте публичные тикеты (Issues) для сообщения о проблемах с безопасностью.

## Модель данных и конфиденциальности
Этот инструмент обрабатывает **журналы ваших собственных взаимодействий с ИИ**. Относитесь к созданному корпусу и файлам аватаров как к **конфиденциальным персональным данным**:
- Корпус (`STUDIE/`), классифицированные фрагменты (chunks) и заполненные файлы аватаров по умолчанию включены в **.gitignore** — никогда не коммитьте реальный корпус.
- Экстрактор удаляет API-ключи, токены, электронную почту, данные, похожие на IP, и длинные последовательности цифр **перед записью**.
- **Оператор несет ответственность за удаление медицинских, налоговых или других конфиденциальных данных пользователя** перед тем, как корпус или любой файл аватара покинет частное окружение.
- Скрипты сами по себе не отправляют никаких данных; классификация запускается через любого агента/LLM, на которого вы укажете — проверяйте обработку данных этим агентом отдельно.

## Секреты
Любой секрет, который когда-либо был закоммичен, должен быть **ротирован** (изменен), а не просто удален из рабочей копии.
