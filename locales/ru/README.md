# build-your-users-mind

> **What you mind is what you get.**

**🌐 [EN](../../README.md) · [DE](../de/README.md) · [ES](../es/README.md) · [JA](../ja/README.md) · [RU](README.md) · [ZH](../zh/README.md)** — Английская версия является авторитетной; переводы могут отставать.

Рецепт для любого ИИ-агента (Claude, Codex, Gemini/agy, Kimi, …) по созданию эмпирической, самообучающейся **theory-of-mind model of its own user** (модели теории разума своего пользователя) на основе собственных журналов взаимодействия, позволяющей действовать в духе пользователя в его отсутствие.

Это работает по принципу **feedforward**: вместо ожидания обратной связи, которая не поступит в отсутствие пользователя, агент *прогнозирует обратную связь от пользователя до её получения* и использует этот прогноз в качестве сигнала руководства/вознаграждения, а затем оценивает прогноз на соответствие реальности для улучшения качества.

## «Я знаю, чего ты хочешь».

Эта фраза выражает всю суть. Агент читает прошлые запросы пользователя, выделяет то, **что пользователь решает, как формулирует это и был ли он удовлетворен**, и превращает это в небольшой набор «живых» документов, к которым он может обратиться в момент принятия решения, когда пользователь недоступен.

Это **не** персонаж чат-бота и **не** громоздкий фреймвок — это метод + несколько скриптов + шаблоны документов. Единственная часть, специфичная для конкретного агента, — это *source adapter* (где каждый агент читает собственные логи). Всё остальное универсально.

## Как это работает — feedback precognition

Цикл выполнения от 0 до 4 (см. `templates/START.md`):

| Шаг | Файл | Роль |
|---|---|---|
| 0 | проект `DECISIONS.md` | приоритет имеют решения, специфичные для проекта (более точные) |
| 1 | `WHAT-<USER>-SAID` | правила/решения, **основанные на фактах** (с цитированием ID запросов) |
| 2 | `WHAT-WOULD-<USER>-SAY` | **precognition** (предчувствие) — прогнозируемая обратная связь + уверенность (🟢/🟡/🔴) |
| 3 | `WHAT-I-DID…` + `MY-ACTIONS.txt` | журнал действий, предпринятых на основе прогноза |
| 4 | `WHAT-<USER>-SAID-ABOUT…` | **оценка** — прогноз против реальности → улучшает (1) и (2) |

Метрика качества = **как часто ожидаемая реакция совпадает с последующей реальной обратной связью от пользователя.**
При 🔴 (новое/нет паттерна) правило гласит: **передавать на следующий уровень, не гадать.**

### Конвейер (построение модели)

1. **Извлечение** (`scripts/corpus_extract.py`) — детерминировано: извлечение только написанных человеком запросов из ваших логов, фильтрация синтетических шагов, **скрытие конфиденциальной информации**, привязка каждого запроса к `outcome_signal` следующего шага (похвала/исправление/повторный запрос/отсутствует).
2. **Разделение (Chunk)** (`scripts/chunk_corpus.py`) — удаление дубликатов, необязательные домены, разделение на фрагменты по размеру для роя.
3. **Классификация** (рой) — таксономия из 8 типов (`TAXONOMY.md`) + decision_kind + паттерн формулировки. Иерархический рой (лидеры доменов × исполнители фрагментов); см. встроенный навык `skills/swarm-operations/`.
4. **Агрегирование** (`scripts/aggregate_stats.py`) — распределение типов, соотношение B:K, точки перелома.
5. **Создание** файлов аватара из `templates/` и **привязка** (bind) короткого указателя в собственный файл памяти/правил агента (Claude `CLAUDE.md`, Codex `GPT.md`/`AGENTS.md`, Gemini `GEMINI.md`, …).

Полное описание см. в `SKILL.md`, а места расположения журналов для каждого агента — в `SOURCE-ADAPTERS.md`.

## Theory of us — теоретическая база

Система моделирует **диаду** (агент ↔ пользователь), а не просто пользователя в изоляции — «theory of us» (теорию о нас).
Она основывается на:
- Исследованиях в области **Theory of Mind** (Модели разума) для LLM-агентов — прогнозирование и учет ментального состояния собеседника улучшает результаты (например, *ToM-SWE*, arXiv 2510.21903; *Infusing Theory of Mind into Socially Intelligent LLM Agents*, 2509.22887; *Persistent Memory & User Profiles*, 2510.07925).
- **Prompt-Archaeology** (L. Geiger) — метод классификации полных протоколов взаимодействия человека и ИИ, чью таксономию из 8 типов повторно использует данный модуль (`TAXONOMY.md`).
- Известное ограничение: модель разума LLM **надежна в повторяющихся случаях, но хрупка при новых/соперничающих вариациях**, отсюда и уровни уверенности, а также правило «передавать на следующий уровень, не гадать».

## Предвзятость и ограничения (прочтите перед тем, как доверять)

- **Молчаливое одобрение невидимо** — пользователи пишут исправления, а не похвалу → модель чрезмерно представляет исправления и искажает картину в сторону «критической». Калибруйте соответствующим образом.
- **Идентификаторы доказательств формируются в результате синтеза LLM** — проверяйте ключевые из них по исходному корпусу.
- **Предвзятость классификатора** — проводите выборочную проверку образцов; для серьезного использования сообщайте о согласии между оценщиками.
