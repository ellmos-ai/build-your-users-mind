# セキュリティポリシー

## 脆弱性の報告
脆弱性を発見した場合は、このリポジトリの **GitHub Private Vulnerability Reporting**（Security → Report a vulnerability）を通じて報告してください。セキュリティ上の問題についてパブリックなIssueを作成しないでください。

## データとプライバシーモデル
このツールは**あなた自身のAI対話ログ**を処理します。生成されたコーパスおよびアバターファイルは**プライベートな個人データ**として扱ってください：
- コーパス（`STUDIE/`）、分類されたフラグメント、および生成されたアバターファイルは、デフォルトで **.gitignore** によって除外されます。実際のコーパスは絶対にコミットしないでください。
- エクストラクターは、書き込みを行う前に、APIキー、トークン、メールアドレス、IPアドレスに類似したもの、および長い数列を隠蔽（マスク）します。
- コーパスやアバターファイルがプライベートな環境から出る前に、健康や税金、その他の機密情報の隠蔽を行うことは**オペレーターの責任**です。
- スクリプト自体が外部へデータを送信することはありません。分類処理は指定されたエージェント/LLMを通じて実行されるため、そのエージェントのデータ処理方針を個別に確認してください。

## 秘密情報（シークレット）
一度でもコミットされた秘密情報は、単にワーキングツリーから削除するだけでなく、必ず**ローテーション（無効化と再生成）**してください。
